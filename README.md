🛰️ مؤخراً واجهتنا مشكلة تقنية في أحد مشاريع الـ LiDAR وكان لازم نحلها بطريقة عملية وقابلة للتنفيذ على كمية ضخمة من البيانات.
كان عندنا آلاف ملفات الـ LAS مرفوعة بنظام:
WGS84 / UTM Zone 38N
وبارتفاعات Ellipsoidal Height.
لكن المطلوب من العميل كان تسليم كل البيانات بالنظام السعودي الرسمي:
GRF17 + KSA Geoid21
وهنا ظهرت المشكلة الحقيقية…
البيانات الأفقية كانت صحيحة، لكن الارتفاعات كانت Ellipsoidal وليست Orthometric، وبالتالي لازم يتم تصحيح الداتم الرأسي بالكامل.
بدل ما تتم المعالجة بشكل يدوي، قررت أبني Workflow كامل باستخدام Python + PDAL.
الخطوات كانت تقريباً كالتالي:
▪️ أول شيء قمت بتجهيز نموذج الجيويد KSA Geoid21.
تم استخراج قيم الجيويد من بوابة GRF17 الرسمية وتحويلها إلى GeoTIFF Raster حتى أستخدمها مباشرة داخل المعالجة.
▪️ بعدها بنيت Pipeline باستخدام PDAL لقراءة ملفات LAS وإعادة الإسقاط من:
EPSG:32638 → EPSG:9358
مع تطبيق تصحيح الارتفاعات باستخدام:
filters.hag_dem
▪️ ثم تم استبدال قيم الـ Z الإهليجية بقيم الارتفاع الأرثومتري الناتجة من نموذج الجيويد.
▪️ وأخيراً تم تشغيل المعالجة بشكل Batch Processing على جميع الملفات تلقائياً.
النتيجة كانت ممتازة:
✅ تحويل كامل إلى GRF17 + KSA Geoid21
✅ معالجة آلاف الملفات بشكل مؤتمت
✅ دقة أفضل للارتفاعات
